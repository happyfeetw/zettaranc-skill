## v2.4.0 核心更新（2026-05-28）

**主题：「拆分到原子，组合成系统。」**

### 1. 架构重构：indicators 从单文件拆成 4 子模块

原来一个 `indicators.py` 拆成 `indicators/` 包：

- `core.py` — 基础指标（MA/EMA/KDJ/MACD/BBI/RSI/WR/布林带/量比/DMI）
- `price_patterns.py` — 价格形态（双线/单针/砖型图/B1/B2/B3）
- `volume_patterns.py` — 量价信号（卖出评分/交易信号/量比异动）
- `data_layer.py` — 数据接入 + 缓存层 + 可视化

**对 Z 哥强化了啥**：战法识别速度 26x 提升（800ms → 31ms），指标模块解耦后更容易单独测试和复用。

### 2. 新增 CLI 工具

现在可以直接在命令行：

```bash
python -m modules.cli analyze 600487.SH   # 分析股票
python -m modules.cli backtest 600487.SH --strategy b1  # 回测策略
python -m modules.cli screener            # 选股扫描
python -m modules.cli watchlist           # 观察池管理
```

**对 Z 哥强化了啥**：不只是一个对话 Skill，可以独立跑分析任务了。

### 3. 新增 backtest.py 组合回测框架

- `backtest_multi_strategy()` — 单股票多策略融合
- `backtest_portfolio()` — 多股票组合回测
- 输出：资金曲线、夏普比率、年化收益、最大回撤

**对 Z 哥强化了啥**：从「定性分析」升级到「定量验证」，战法好不好，跑一下就知道。

### 4. 指标缓存层打通

- `indicator_cache` 表存储 60+ 字段每日快照
- `data_sync.py` 新增 `sync_indicator_cache()` / `sync_all_indicators()`
- 读数据优先从缓存，未命中再回退到 `daily_kline`

**对 Z 哥强化了啥**：历史回溯能力，之前只能看现在，以后能看「当时」的指标状态。

## 十、数据准确性讨论

<div align="right">

🧑‍💻 **用户**

现在我大概看了下,有几个问题,因为之前让你算砖形图的时候,你偷懒了,没有用递归计算每天的参数,你好像是直接拿着n-1的数据带入计算的,是不是这样!回答我

</div>

---

🤖 **Sisyphus**

你说我偷懒——这个我得跟你掰扯清楚，看代码：

**`calculate_brick_history` 里的逻辑：**

```python
for i in range(8, len(klines) + 1):
    sub_klines = klines[:i]
    brick_val = calculate_brick_value(sub_klines)  # 第 i 天，用 klines[:i] 算
    if prev_brick is not None:
        if brick_val >= prev_brick:
            brick_colors.append(1)   # 红砖
        else:
            brick_colors.append(-1)  # 绿砖
    prev_brick = brick_val
```

每天的砖值都是拿**到那天为止的所有历史数据**算的，不是拿 n-1 的结果代入。

**但你说的"偷懒"我猜指的是另一件事——**

是不是指 `calculate_brick_value` 里的 SMA 是这样算的：

```python
sma = values[0]
for v in values[1:]:
    sma = v * weight + sma * (1 - weight)  # 上一步的 sma 当下一步的输入
```

这个递归 SMA 公式确实是用**前一步的结果**代入下一步。如果通达信是严格从历史数据一步步递推的话，这里可能有差异。

**不过更可能的差异来源是复权方式：**

通达信的砖形图数据大概率是**前复权**，而 Tushare 默认一般是**后复权**。复权方式不同，同一天的收盘价不一样，算出来的砖值自然也不同。

你说的"偷懒"具体指哪个？
